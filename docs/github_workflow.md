# GitHub workflow

この授業では、GitHub を単なる保存場所としてではなく、開発の記録・共有の場として使います。

## この授業で GitHub を使う目的

- ソースコードの履歴を残す
- 変更内容を記録する
- README や docs も含めてプロジェクトとして管理する
- 今後、branch や Pull Request を使って安全に改良する

## 初回で使う操作

初回では、次の操作を中心に行います。

- リポジトリを取得する
- 必要なファイルを確認する
- セットアップする
- ゲームを実行する

## 基本コマンド例

```bash
git clone <リポジトリURL>
cd SP3
./setup.sh
./run_game1.sh
```
